## 1. 제품 개요

### 1.1 배경

임직원의 자기개발 학습 계획과 수행 내역이 문서, 설문, 메신저 등에 분산되어 있어 개인은 계획 대비 진척도를 파악하기 어렵고, 리더와 HRD는 필요한 지원 시점과 조직의 학습 현황을 적시에 확인하기 어렵다.

### 1.2 제품 정의

임직원이 자기개발 목표와 학습 계획을 수립하고 수행 내역을 기록하면, 개인·팀·전사 관점에서 계획 대비 진척도와 학습 추이를 시각화하는 사내 웹 대시보드다.

### 1.3 해결하려는 핵심 문제

1. 개인이 계획 대비 실제 수행 수준을 한눈에 알기 어렵다.
2. 학습 기록 방식이 일관되지 않아 데이터 집계와 비교가 어렵다.
3. 리더가 팀원의 학습 정체 여부와 지원 필요성을 조기에 파악하기 어렵다.
4. HRD가 학습 제도와 예산의 활용 현황을 근거 기반으로 평가하기 어렵다.

### 1.4 제품 원칙

- 기록보다 성장을 지원한다.
- 개인 데이터는 필요한 범위에서만 공개한다.
- 단순 학습 시간보다 계획의 지속성과 달성도를 함께 본다.
- 순위 경쟁보다 목표 달성과 지원이 필요한 신호를 강조한다.

## 2. 목표와 범위

### 2.1 제품 목표

- 임직원이 분기 학습 계획을 10분 이내에 등록할 수 있다.
- 임직원이 주간 학습 수행 내역을 1분 이내에 기록할 수 있다.
- 개인이 계획 대비 진행률과 남은 학습량을 즉시 확인할 수 있다.
- 리더가 팀의 학습 현황과 지원 필요 대상을 한 화면에서 확인할 수 있다.
- HRD가 조직별 참여율, 달성률, 학습 분야 및 추이를 분석할 수 있다.

### 2.2 비목표

MVP에서는 다음 기능을 제공하지 않는다.

- 강의 영상 재생 및 콘텐츠 직접 호스팅
- AI 기반 학습 콘텐츠 추천
- 임직원 공개 랭킹 및 보상 자동 지급
- 학습 결과와 인사평가 점수의 자동 연계
- 외부 학습 플랫폼과의 실시간 양방향 연동
- 네이티브 모바일 앱

### 2.3 MVP 범위

- 앱 내 사용자 직접 등록 및 프로필 선택
- 분기별 학습 목표와 월·주 단위 계획 등록
- 학습 수행 내역 직접 등록 및 수정
- 개인, 팀 리더, HRD 대시보드
- 기본 필터와 CSV 다운로드
- 주간 기록 리마인드 및 계획 지연 알림
- 역할 기반 접근 제어와 감사 로그

## 3. 사용자와 권한

| 사용자 | 주요 목적 | 조회 범위 |
|---|---|---|
| 임직원 | 목표 수립, 수행 기록, 진척 확인 | 본인 데이터 |
| 팀 리더 | 팀 현황 확인, 지원 대상 파악 | 직속 조직의 요약 및 구성원별 현황 |
| HRD 담당자 | 제도 운영, 조직별 분석, 리포트 작성 | 전사 데이터 및 조직별 집계 |
| 시스템 관리자 | 계정, 권한, 코드 및 시스템 운영 | 운영에 필요한 시스템 정보 |
| 경영진 | 전사 학습 현황 확인 | 전사·조직 단위 집계 정보 |

### 3.1 공개 원칙

- 학습 메모와 증빙 자료는 본인 및 명시적으로 허용된 관리자만 조회할 수 있다.
- 리더는 팀원의 목표명, 계획 시간, 수행 시간, 진행 상태를 조회할 수 있다.
- 다른 임직원 간 개별 학습 현황과 순위는 공개하지 않는다.
- 경영진 화면은 개인 식별 정보 없이 조직 단위 집계로 제공한다.
- 인사평가에 활용할 경우 별도의 정책 고지와 동의 절차를 마련한다.

## 4. 핵심 사용자 여정

### 4.1 계획 수립

1. 임직원이 분기를 선택한다.
2. 학습 목표명, 목적, 분야, 기간 및 목표치를 입력한다.
3. 목표 달성을 위한 월별 또는 주별 계획을 등록한다.
4. 시스템이 총계와 기간의 정합성을 검증한다.
5. 등록 완료 후 개인 대시보드에 계획이 반영된다.

### 4.2 수행 기록

1. 임직원이 진행 중인 목표를 선택한다.
2. 학습일, 활동 유형, 수행 시간, 진척도 및 메모를 입력한다.
3. 필요 시 링크 또는 증빙 자료를 첨부한다.
4. 저장 즉시 누적 수행량과 계획 대비 달성률이 갱신된다.

### 4.3 진척 확인 및 조정

1. 임직원이 개인 대시보드에서 현재 진척도를 확인한다.
2. 지연 목표의 원인과 남은 필요 수행량을 확인한다.
3. 계획을 수정하거나 목표를 완료·보류 처리한다.
4. 시스템은 계획 변경 이력을 기록한다.

### 4.4 팀 지원

1. 리더가 기간과 상태를 선택해 팀 현황을 조회한다.
2. 미참여, 지연, 장기 미기록 구성원을 확인한다.
3. 구성원과 면담하거나 필요한 시간·예산·자료를 지원한다.
4. 리더가 개입 내용을 선택적으로 메모한다.

## 7. 지표 정의

모든 비율은 선택한 기간과 필터 조건을 기준으로 계산한다.

| 지표 | 정의 |
|---|---|
| 계획 등록률 | 학습 목표를 1개 이상 등록한 대상 인원 ÷ 전체 대상 인원 × 100 |
| 학습 참여율 | 수행 내역을 1건 이상 등록한 인원 ÷ 전체 대상 인원 × 100 |
| 시간 달성률 | 누적 수행 시간 ÷ 계획 시간 × 100 |
| 목표 완료율 | 완료 처리된 목표 수 ÷ 종료일이 도래한 목표 수 × 100 |
| 계획 준수율 | 계획한 주차에 실제 수행이 발생한 주차 수 ÷ 계획 주차 수 × 100 |
| 활성 학습자 | 최근 30일 내 수행 내역을 1건 이상 등록한 사용자 |
| 장기 미기록 | 진행 중인 목표가 있으나 최근 14일간 수행 기록이 없는 상태 |
| 지연 목표 | 기준일 기대 진척도보다 실제 진척도가 20%p 이상 낮은 목표 |

### 7.1 진행률 계산 원칙

- 시간형 목표: `누적 수행 시간 ÷ 계획 시간 × 100`
- 진척도형 목표: 사용자가 입력한 최신 누적 진척도
- 화면 표시 상한은 100%로 하되 초과 수행 시간은 별도 값으로 보존한다.
- 기대 진척도는 목표 기간의 경과 비율을 기본값으로 사용한다.

## 8. 데이터 요구사항

### 8.1 핵심 데이터 객체

| 객체 | 주요 필드 |
|---|---|
| 사용자 | 사번, 이름, 이메일, 조직, 직군, 직급, 재직 상태 |
| 학습 목표 | 목표 ID, 사용자 ID, 목표명, 목적, 분야, 기간, 측정 방식, 목표치, 상태 |
| 세부 계획 | 계획 ID, 목표 ID, 기준 주·월, 계획 시간 또는 계획 진척도 |
| 수행 내역 | 활동 ID, 목표 ID, 학습일, 유형, 수행 시간, 진척도, 메모 |
| 첨부 자료 | 파일 ID, 활동 ID, 파일명, 저장 위치, 등록일 |

### 8.2 데이터 품질 규칙

- 퇴사자 데이터는 보존 정책에 따라 비활성 처리하고 신규 집계 대상에서 제외한다.
- 조직 이동 시 과거 데이터는 활동 당시 조직 기준을 유지한다.
- 동일 날짜의 동일 활동이 중복 등록될 경우 사용자에게 확인 메시지를 표시한다.
- 삭제는 원칙적으로 논리 삭제하며 감사 로그를 유지한다.
- 집계 데이터는 원천 데이터와 일 단위로 대조할 수 있어야 한다.

## 12. 수용 기준

### 12.1 개인 사용자

- 사용자가 목표와 주별 계획을 등록하면 개인 대시보드에 즉시 반영된다.
- 수행 내역 저장 후 계획 대비 달성률이 올바르게 재계산된다.
- 계획이 없는 기간에는 0%가 아닌 ‘계획 없음’으로 표시된다.
- 권한이 없는 사용자는 다른 사용자의 상세 학습 내역에 접근할 수 없다.

### 12.2 팀 리더

- 리더는 현재 소속된 직속 구성원의 현황만 조회할 수 있다.
- 구성원별 최근 기록일과 지원 필요 상태가 지표 정의와 일치한다.
- 조직 이동자의 과거 기록은 당시 조직 집계에 남는다.

### 12.3 HRD

- 조직, 기간, 직군, 분야 필터가 KPI·차트·표에 동일하게 적용된다.
- 최소 집계 인원 미만 그룹은 개인을 추정할 수 없도록 마스킹된다.
- 화면 집계와 CSV 다운로드 결과의 수치가 일치한다.
